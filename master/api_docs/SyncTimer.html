Class: SyncTimer

SyncTimer

new SyncTimer(report)

This timer synchronizes to requesAnimationFrame so that all callbacks are performed right after a refresh took place. When setting a timeout it atempts to predict which refresh will take place nearest to the timeout.
Parameters:
Name Type Description
report function Reporting function for reporting timing info

setTimeout(timeout, callbackDraw, callbackDone, name)

Set a timeout; callback is called on next requestFrame that is as near as possible to the timeout.
Parameters:
Name Type Description
timeout int Number of ms until callbackDone is called, use multiples of 16 for best results (since most screens have a 60hz refresh rate). If -1 then never tmieout
callbackDraw function Callback called when timeout starts (immediately if we handling a synchronous callback, at next refresh if not)
callbackDone function Callback called on timeout
name String Name of this timeout for logging. Default = noname
